git clone <ссылка из гитхаб> - клонирование репозитория на ПК
git status - сверить с гитом, проверить, какие файлы не отслеживаются гитом, есть только на локале
git add . - если хотим ослеживать что-то гитом, начнет отслеживать все файлы, которые ранее не отслеживались
git add <имя файла> - если хотим добавить на ослеживание какой-то один файл
git commit -m "комментарий" - закоммитить на гит, сделать "фото" репозитория на ПК
git push - отправить коммит на внешний репозиторий
git commit -am "коммент" - сразу закоммитить и отправить изменения, после git push
git fetch - проверить есть ли изменения на внешнем репозитории
git pull - скачать состояние внешнего репозитория на локальный
git commit, потом git pull (проверить есть ли изменения на внешнем репозитории), потом git push - если работают несколько человек, чтобы не возникли конфликты
